# 播客制作流水线

你有一个播客创意，甚至可能有一堆话题 backlog。但在研究嘉宾、撰写大纲、起草开场白、生成节目笔记以及撰写社交媒体推广帖子之间——制作开销扼杀了你的动力。如果你提交一个话题，然后得到一个完整的制作包会怎样？

这个用例将智能体链接在一起，处理从话题到可发布素材的整个播客制作工作流程。

## 痛点

单打独斗的播客主和小团队花在制作上的时间比实际录制还多。研究需要数小时，节目笔记是事后考虑，社交媒体推广是第一个被跳过的事情。创意部分——对话——可能只占总努力的30%。这个智能体处理另外70%。

## 它做什么

- **节目研究**——给定话题或嘉宾姓名，编译背景研究、谈话要点和建议问题
- **大纲与脚本**——生成结构化的节目大纲，包括开场脚本、段落过渡和结束语
- **节目笔记**——录制后，将转录稿处理成带时间戳的节目笔记，包含提到的所有内容的链接
- **社交媒体套件**——为 X、LinkedIn 和 Instagram 创建推广帖子，包含节目亮点和引用语录
- **节目描述**——为 Spotify、Apple Podcasts 和 YouTube 撰写 SEO 优化的节目描述

## 你需要的技能

- 网页搜索 / 研究技能（用于嘉宾研究和话题深度挖掘）
- 文件系统访问（用于读取转录稿和写入输出文件）
- Slack、Discord 或 Telegram 集成（用于交付素材）
- 可选：`sessions_spawn` 用于并行运行研究和写作智能体
- 可选：RSS 订阅技能（用于监控竞争对手播客）

## 如何设置

1. 录制前——生成研究和outline：
```text
我要录制一集关于[话题]的播客。我的嘉宾是[姓名]。

请：
1. 研究嘉宾——他们的背景、最近的工作、热门观点，以及他们在公开场合说过的任何有争议或有趣的话。
2. 研究话题——关键趋势、最近的新闻、常见误解，以及听众可能已经知道的与会令他们惊讶的内容。
3. 生成节目大纲：
   - 开场钩子（1-2句话吸引注意力）
   - 开场脚本（30秒，随意语气）
   - 5-7个采访问题，从简单/建立关系到深入/发人深省排序
   - 2-3个"备用"问题，以防对话陷入僵局
   - 带有行动号召的结束环节

将所有内容保存到 ~/podcast/episodes/[episode-number]/prep/
```

2. 录制后——生成节目笔记和推广：
```text
这是第[NUMBER]集的转录稿：[粘贴或指向文件]

请：
1. 撰写带时间戳的节目笔记——每个主要话题转换都获得时间戳和一句话摘要。包含提到的任何内容的链接（工具、书籍、文章、人）。
2. 撰写节目描述（最多200字），针对播客搜索优化。自然包含3-5个相关关键词。
3. 创建社交媒体帖子：
   - X/Twitter：3条推文——一条引用语录、一条关键洞察、一个引发讨论的问题。每条少于280字。
   - LinkedIn：1条帖子，专业语气，100-150字。
   - Instagram 标题：1条帖子带表情符号，随意语气，包含相关标签。
4. 提取"亮点"列表——3个最有趣/令人惊讶的时刻及其时间戳。

将所有内容保存到 ~/podcast/episodes/[episode-number]/publish/
```

3. 可选——竞争对手监控：
```text
每天监控这些播客 RSS 订阅：
- [feed URL 1]
- [feed URL 2]

当一集涉及与我播客相关的话题的新节目发布时，给我发送 Telegram 消息，包含：
- 节目标题和链接
- 一句话摘要
- 这是我应该回应的，还是应该从我的角度涵盖的内容
```

## 关键洞察

- **录制前的研究**是价值最大的地方。带着深入的嘉宾研究进入采访会让对话显著更好——这是你在后期制作中无法伪造的。
- 带时间戳的节目笔记是一个巨大的听众留存工具。大多数播客主跳过它们因为它们很乏味。这个智能体让它们变得毫不费力。
- 社交媒体套件节省了最多的*经常性*时间。你需要为每集节目做推广，而且结构总是相同的——非常适合自动化。
- 与[多智能体内容工厂](content-factory.md)用例配对，如果你想将播客内容重新用于博客文章、新闻通讯或视频剪辑。
