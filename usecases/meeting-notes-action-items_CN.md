# 自动会议记录与行动项

你刚结束了一场45分钟的团队通话。现在你需要撰写总结、提取行动项，并将它们分发到 Jira、Linear 或 Todoist——手动操作。当你完成时，下一场会议已经开始了。如果你的智能体在转录稿一落地就处理完所有这些呢？

这个用例将任何会议转录稿转换为结构化笔记，并自动在你的项目管理工具中创建任务。

## 痛点

会议记录乏味但关键。大多数人要么跳过它们（然后失去上下文），要么花费20多分钟撰写。行动项会被遗忘，因为它们存在于某人的脑海中或埋在聊天线程中。这个智能体消除了"我们讨论过它"和"它已被追踪和分配"之间的差距。

## 它做什么

- **监控**新的会议转录稿（通过 Otter.ai 导出、Google Meet 转录、Zoom 录制摘要，或直接粘贴到聊天中）
- **提取**关键决策、讨论主题以及带有负责人和截止日期的行动项
- **在 Jira、Linear、Todoist 或 Notion 中创建任务**——分配给正确的人，并带有会议上下文
- **将摘要发布**到 Slack 或 Discord，以便整个团队都有记录
- **跟进**——可选地通过定时提醒在截止日期前 ping 分配人

## 你需要的技能

- Jira、Linear、Todoist 或 Notion 集成（用于创建任务）
- Slack 或 Discord 集成（用于发布摘要）
- 文件系统访问（用于读取转录文件）
- 调度 / cron（用于跟进提醒）
- 可选：Otter.ai、Fireflies.ai 或 Google Meet API 用于自动获取转录稿

## 如何设置

1. 选择你的转录稿来源。最简单的方法是直接将转录稿粘贴到聊天中。要实现自动化，设置文件夹监控或 API 集成。

2. 提示 OpenClaw：
```text
我刚结束了一场会议。这是转录稿：

[粘贴转录稿或指向文件]

请：
1. 撰写简洁的摘要（最多5个要点），涵盖关键决策和主题。
2. 提取所有行动项。对于每一项，识别：
   - 需要做什么
   - 谁负责（将姓名匹配到我的团队）
   - 截止日期（如果提到，否则标记为"待定"）
3. 为每个行动项创建 Jira 工单，分配给正确的人。
4. 将完整摘要发布到 Slack 的 #meeting-notes 频道。
```

3. 完全自动化流水线（转录文件夹监控）：
```text
设置一个重复任务：每30分钟检查 ~/meeting-transcripts/ 中新的 .txt 或 .vtt 文件。当你找到一个时：

1. 将转录稿解析为带有行动项的结构化摘要。
2. 为每个行动项在 Linear 中创建任务。
3. 将摘要发布到 Slack 的 #team-updates 频道。
4. 将处理过的文件移动到 ~/meeting-transcripts/processed/。

对于每个有截止日期的行动项，设置一个提醒，在截止日期前一天在 Slack 中 ping 分配人。
```

4. 自定义输出格式：
```text
撰写会议摘要时，始终使用以下结构：
- 顶部是**日期与参会者**
- **关键决策**——已决定事项的编号列表
- **行动项**——表格，列：任务、负责人、截止日期、状态
- **未决问题**——任何需要跟进的未解决事项
```

## 关键洞察

- 真正的价值不在于摘要——而在于**自动任务创建**。没有变成已追踪任务的会议记录只是文档表演。
- 与[Todoist 任务管理器](todoist-task-manager.md)用例配对，以完全了解智能体创建的任务。
- 来自 Zoom 或 Google Meet 的 VTT/SRT 字幕文件非常适合作为输入——它们包含时间戳，有助于智能体将陈述归因于说话者。
- 从简单开始（粘贴转录稿，获取摘要）并逐步自动化。在验证输出质量之前不要过度设计流水线。
