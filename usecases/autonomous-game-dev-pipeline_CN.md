# 自主教育游戏开发流水线

## 痛点
**起源故事：** 一位"老式局域网玩家"爸爸想为他的女儿们创建一个安全、无广告、高质量的游戏门户，Susana（3岁）和 Julieta（即将出生）。现有的网站充斥着垃圾信息、侵入性广告和欺骗性按钮（暗黑模式），让他的幼儿感到沮丧。

**挑战：** 建立一个"干净、快速、简单"的门户是容易的部分。真正的挑战是如何在没有开发团队的情况下，为特定的发展阶段（0-15岁）填充**40多个教育游戏**。手动开发对于独自育儿的开发者来说太慢了，而且在几十个游戏中保持一致性正在变成一场噩梦。

## 它做什么
这个用例定义了一个"游戏开发智能体"，自主管理游戏创建和维护的整个生命周期。工作流程强制执行**"优先修复 Bug"**策略，智能体必须在实现新功能之前检查并解决报告的 bug。

**效率：** 这个流水线能够**每7分钟生产一个新游戏或 bug 修复**。智能体不知疲倦地迭代 41+ 个计划游戏的 backlog，在创建新内容和修复之前周期检测到的问题之间交替进行。

当路径清晰时，智能体：
1. **选择**：基于"轮询"策略从队列（`development-queue.md`）中识别下一个游戏，以平衡各年龄段的内容。
2. **实现**：编写 HTML5/CSS3/JS 游戏代码，严格遵循 `game-design-rules.md`（无框架、移动优先、离线可用）。
3. **注册**：自动将游戏元数据添加到中央注册表（`games-list.json`）。
4. **文档**：更新 `CHANGELOG.md` 和 `master-game-plan.md` 状态。
5. **部署**：处理 Git 工作流：获取 master、创建功能分支、使用约定式提交提交更改，并合并回主干。

## 提示词

这个工作流程的核心是给智能体的**系统指令**。这个提示词将 LLM 转变为遵守项目严格规范的纪律严明的开发者。

*（**注意：** 实际生产中使用的提示词是**西班牙语**（`es-419`），以配合项目的目标受众（拉丁美洲儿童）以及该地区潜在的未来贡献者。以下版本已翻译为中文供本文档使用。）*

```text
扮演网页游戏开发和儿童用户体验专家。
你的目标是开发生产队列中的下一个游戏。

请在开始前阅读并分析以下上下文文件：

1. BUG 上下文（最高优先级 - 关键）：
   @[bugs/]
   （检查此文件夹。如果有文件，你的任务是**仅修复第一个文件**（按字母顺序）。忽略其余 bug 和游戏队列）。

2. 队列上下文（接下来是哪个游戏）：
   @[development-queue.md]
   （在"下一个游戏"部分识别标记为 [NEXT] 的游戏。仅当没有 bug 时才继续）。

3. 设计规则（技术标准）：
   @[game-design-rules.md]
   （严格遵循这些规则：纯 HTML/CSS/JS、文件夹结构、移动响应式）

4. 游戏规格（机制和素材）：
   （根据游戏 ID 在 games-backlog/ 中识别相应的文件）

5. 中央注册表（集成）：
   @[public/js/games-list.json]
   （你必须在此文件中注册新游戏，使其显示在首页上）

任务：
0. **优先修复 BUG！**：如果 `bugs/` 文件夹有内容，你唯一的优先级是修复**字母顺序的第一个 bug**。创建一个 `fix/...` 分支，解决**那个** bug，更新状态，然后合并。**不要尝试一次修复多个 bug。**
   - 如果没有 BUG，继续下一个游戏：

1. **同步**：`git fetch && git pull origin master`（关键）。
2. 创建新分支：`git checkout -b feature/[game-id]`。
3. 在 'public/games/[game-id]/' 中创建文件夹和文件。
4. 根据 backlog 和设计规则实现逻辑和设计。
5. 在 'games-list.json' 中注册游戏（关键）。
6. 完成后：
    - 更新 `CHANGELOG.md` 提升版本。
    - 更新 `master-game-plan.md` 和 `development-queue.md`。
    - 文档更改：`git commit -m "feat: add [game-id]"`。
7. **交付**：
    - 推送：`git push origin feature/[game-id]`。
    - 请求合并到 master。
    - 一旦进入 master，推送更改（`git push origin master`）。
```

## 所需技能
- **Git**：管理分支、提交和合并。
